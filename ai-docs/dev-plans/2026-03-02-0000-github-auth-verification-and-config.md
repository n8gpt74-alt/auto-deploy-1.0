# План: Настройка и проверка авторизации через GitHub

## Цель
Обеспечить возможность входа любого пользователя через GitHub OAuth. 
В репозитории уже реализован необходимый код для `next-auth` (включая провайдера GitHub, middleware и страницы входа/выхода). Основная задача — выполнить финальную настройку окружения и проверить работоспособность флоу.

## Состояние проекта (подтверждено)
- **Библиотеки**: `next-auth` ^4.24.13 установлена.
- **Конфигурация**: `src/lib/auth.ts` содержит `GitHubProvider` с запрашиваемыми правами `read:user user:email repo`.
- **Маршруты**: 
    - `/api/auth/[...nextauth]` настроен.
    - `/login` и `/logout` созданы.
    - `/dashboard` защищен через `middleware.ts`.
- **UI**: Кнопки "Login with GitHub" присутствуют на главной странице и странице логина.

## Риски
- **Callback URL**: Ошибка в `Authorization callback URL` на стороне GitHub приведет к ошибке `redirect_uri_mismatch`. 
    - *Решение*: Убедиться, что URL в GitHub точно совпадает с `http://localhost:3000/api/auth/callback/github`.
- **Scopes**: Если не запрашивать `repo`, список репозиториев на дашборде будет пустым для приватных репо.
    - *Решение*: В `src/lib/auth.ts` уже указано `scope: "read:user user:email repo"`.

## Проверка готовности (Acceptance Criteria)
1. Пользователь может успешно авторизоваться через GitHub.
2. Сессия сохраняется, и пользователь получает доступ к защищенным роутам (`/dashboard`).
3. Информация о пользователе (имя, email, аватар) корректно отображается в приложении (если предусмотрено UI).
4. Пользователь может выйти из системы через `/logout`.
